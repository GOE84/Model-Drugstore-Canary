# Model-Drugstore-Canary**PDPA**

The "Drugstore Canary" (ระบบเตือนภัยโรคระบาดจากยอดขายยา)

แนวคิด: ก่อนที่คนจะไปโรงพยาบาลรัฐเพื่อแจ้งว่าเป็นโรค ผู้ป่วยส่วนใหญ่จะเดินเข้า ร้านขายยา หรือ 7-11 เพื่อซื้อยาแก้ปวด ยาแก้แพ้ หรือผงเกลือแร่ก่อนเสมอ ยอดขายยาเหล่านี้คือ "สัญญาณเตือนภัยล่วงหน้า" (Early Warning Signal) ที่ไวกว่าสถิติของกระทรวงสาธารณสุข 1-2 สัปดาห์

กลุ่มเป้าหมาย (Customer): เครือข่ายร้านขายยา (เช่น Fascino, Boots, ร้านยาท้องถิ่น), บริษัทยา (Suppliers), โรงพยาบาลเอกชน

ML Model & Technique:

Time-Series Anomaly Detection: ใช้โมเดลจับความผิดปกติของยอดขายในแต่ละพื้นที่

Algorithm: Prophet (ของ Facebook) หรือ LSTM (Long Short-Term Memory)

Input Features: ยอดขายยารายวัน (แยกประเภท: แก้ไข้, แก้ท้องเสีย, ยาทำแผล), ตำแหน่งร้านยา, ข้อมูลน้ำท่วม (ความสูง/ระยะเวลาที่ท่วมขัง)

การทำงาน:

โมเดลเรียนรู้ Pattern ยอดขายปกติของร้านยาแต่ละโซนในหาดใหญ่

เมื่อน้ำลด ถ้ายอดขาย "ยารักษาโรคน้ำกัดเท้า" หรือ "เกลือแร่" (ท้องเสีย) พุ่งสูงขึ้นผิดปกติในโซน A

ระบบจะแจ้งเตือน (Alert) ว่า "โซน A กำลังจะมีการระบาดของโรคผิวหนัง/ท้องร่วง"
